## 一、你的独特优势

基于你的描述，AgentCenter 拥有独特的起点：

| 你的资产 | 意义 |
|---------|------|
| **企业内部全套工具平台** | 需求、设计、代码、检查、构建、测试、CICD — 完整链路 |
| **信息气泡系统** | 已有主动推送能力，不是从零开始 |
| **网页对话框** | 已有对话入口，不是从零开始 |
| **MCP/Skill 生态** | OpenClaw 已有成熟的工具集成方案 |

> **关键洞察**：你不是从零构建一个聊天机器人，而是**给现有的工具链加上智能编排层**。

## 二、业界趋势（2026）

| 趋势 | 说明 | 对你的启示 |
|------|------|-----------|
| **Agentic Workflow** | AI 从"帮你做"变成"替你做" | 对话不只是查询，要能触发执行 |
| **Ambient Intelligence** | AI 主动监控、主动冒泡 | 你的气泡系统是正确方向 |
| **Multi-Agent Orchestration** | 多智能体协作分工 | 需求、设计、代码检查可以是不同 Agent |
| **Human-in-the-Loop** | AI 提议，人确认后执行 | 关键操作需要审批节点 |
| **Outcome Metrics** | 不看 MAU，看任务完成率 | 要有完成率、响应时间等指标 |

## 三、AgentCenter 能做到的事情

### 3.1 核心能力矩阵

| 能力层级 | 能力 | 说明 |
|---------|------|------|
| **L1: 信息聚合** | 统一搜索 | 一次搜索，跨所有平台结果 |
| | 状态总览 | 所有工具平台的关键状态，一个界面看清 |
| | 上下文继承 | 对话框记住当前项目/需求/分支 |
| **L2: 对话触发** | 自然语言操作 | "帮我创建需求"、"帮我触发构建" |
| | 多步骤编排 | "从需求生成设计稿并通知 PM" |
| | 跨平台协调 | 读 Jira → 写 Confluence → 通知 Slack |
| **L3: 主动冒泡** | 异常监控 | CICD 失败、代码检查告警 → 主动推送 |
| | 进度同步 | 长流程 (需求→设计→开发→测试) 自动节点通知 |
| | 智能提醒 | 到期未完成、阻塞等待、依赖超时 |
| **L4: 自主执行** | 自动化流水线 | 对话描述 → 自动执行完整流程 |
| | 多智能体协作 | 需求 Agent + 设计 Agent + 代码 Agent 分工 |
| | 审批流 | AI 提议 → 人确认 → 自动执行 |

## 五、与 SDD 框架的关系

### 5.1 SDD 框架是 AgentCenter 的"技能商店"

AgentCenter 是一个 **SDD 框架无关** 的平台底座，任何 SDD 框架只需实现适配器接口即可接入。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    SDD Framework Adapter Layer                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌───────────────────────────────────────────────────────────────────┐   │
│   │                    AgentCenter Skill Router                          │   │
│   │                                                                       │   │
│   │   用户: "/spec 从需求生成设计稿"                                      │   │
│   │               ↓                                                      │   │
│   │   路由到: Spec-Kit Adapter                                          │   │
│   │               ↓                                                      │   │
│   │   执行: specify → plan → tasks → implement                          │   │
│   │               ↓                                                      │   │
│   │   结果: 设计稿已生成，同步到设计平台                                   │   │
│   └───────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   框架可插拔：                                                            │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐               │
│   │Spec-Kit  │  │OpenSpec  │  │Sisyphus  │  │  自定义  │  ...           │
│   │Adapter   │  │Adapter   │  │Adapter   │  │Adapter   │               │
│   └──────────┘  └──────────┘  └──────────┘  └──────────┘               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 框架无关的 Skills

```yaml
# AgentCenter Skill: requirement-to-design
name: requirement-to-design
description: |
  将需求转换为设计制品
  Trigger: "转设计", "生成设计稿", "需求转接口"
  
framework_agnostic: true  # 不绑定特定 SDD 框架

required_mcp:
  - requirement_platform   # 读取需求
  - design_platform        # 写入设计

  1. 获取需求详情
  2. 生成架构设计 (LLM)
  3. 生成接口设计 (OpenAPI)
  4. 生成 ER 图
  5. 写入设计平台
  6. 通知相关人
```

## 六、竞争差异

| 维度 | 传统方案 | AgentCenter |
|------|---------|-------------|
| **入口统一** | 各平台分散跳转 | 对话一个入口 |
| **操作效率** | 人工切换 + 复制粘贴 | 一句话触发多步骤 |
| **信息聚合** | 手工汇总各平台状态 | 统一搜索 + 状态总览 |
| **主动推送** | 各平台独立告警 | 智能归并 + 上下文 |
| **SDD 集成** | 绑定某一框架 | 框架无关，任意插拔 |
| **扩展性** | 新平台需改造 | MCP 适配器即插即用 |

## 七、实施路线图建议

### Phase 1: 对话触发 (0-3 个月)
- ✅ 对话解析：自然语言 → 工具调用
- ✅ MCP 集成：3-5 个核心平台 (需求、设计、代码检查)
- ✅ 状态查询：跨平台统一搜索
- ✅ 基础气泡：关键告警冒泡

### Phase 2: 流程编排 (3-6 个月)
- ✅ 多步骤编排：从需求 → 设计稿 的完整流程
- ✅ Human-in-the-Loop：关键节点人工确认
- ✅ SDD 适配器：Spec-Kit / OpenSpec / Sisyphus
- ✅ 通知集成：邮件、Slack、飞书等

### Phase 3: 智能协作 (6-12 个月)
- ✅ 多智能体：需求 Agent + 设计 Agent + 检查 Agent 协作
- ✅ 主动监控：全链路流程状态跟踪 + 异常预警
- ✅ 自动化流水线：对话描述 → 完整执行
- ✅ 指标仪表盘：完成率、响应时间、满意度
