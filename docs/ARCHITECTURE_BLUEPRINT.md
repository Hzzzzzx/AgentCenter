## 三、Agent 与外部系统的交互模式

### 3.1 感知模式（Agent 查询外部状态）

```
用户问：帮我看看 PROJ-123 现在在哪个阶段？

AgentCenter                              外部系统
    │                                        │
    │──查询──▶ Jira API                      │
    │           GET /issue/PROJ-123         │
    │                                        │
    │◀──状态──┘                              │
    │   "设计中，已进行3天"                   │
    │                                        │
Agent 理解后回答：
"需求 PROJ-123 目前在『设计中』阶段，已进行 3 天，
  当前等待 API 设计评审，负责人是 @张三"
```

### 3.2 触发模式（Agent 驱动外部流程）

```
用户说：帮我把这个需求转成设计稿

AgentCenter                              外部系统
    │                                        │
    │──查询需求──▶ Jira API                  │
    │                                        │
    │──创建设计文档──▶ Confluence API        │
    │                                        │
    │──通知PM──▶ 飞书 API                   │
    │                                        │
Agent 返回：
"已完成：
  ✅ 设计稿已创建：DOC-2024-1023
  ✅ 已通知 @PM
  ✅ 关联到需求 PROJ-123"
```

### 3.3 合规检查模式（Agent 确保合规）

```
用户说：帮我部署到生产

AgentCenter                              外部系统
    │                                        │
    │──检查变更窗口──▶ 外部合规系统           │
    │                                        │
    │──检查审批状态──▶ ArgoCD/Jira          │
    │                                        │
    │──检查SLO/Error Budget──▶ 监控系统     │
    │                                        │
    │◀──检查结果──┘                         │
    │   "当前不在变更窗口内（允许时间：周末） │
    │    审批状态：待 Ops 负责人审批"        │
    │                                        │
Agent 返回：
"部署到生产需要满足：
  ❌ 当前不在变更窗口内
  ⏳ 待 Ops 负责人审批
  ✅ Error Budget 充足
  ✅ 无高危告警

需要我帮你预约变更窗口吗？"
```

## 四、Agent 视角需要的能力

### 4.1 感知能力

| 能力 | 说明 |
|------|------|
| **状态查询** | 能查询外部系统状态（工单/审批/部署/告警） |
| **上下文构建** | 能整合用户、项目、系统多个维度的上下文 |
| **时间感知** | 知道当前时间、变更窗口、deadline |

### 4.2 触发能力

| 能力 | 说明 |
|------|------|
| **API 调用** | 能调用外部系统 API 执行操作 |
| **流程触发** | 能提交审批、触发部署、发送通知 |
| **批量操作** | 能并行处理多个任务 |

### 4.3 合规感知

| 能力 | 说明 |
|------|------|
| **规则检查** | 能检查变更窗口、审批状态、必过门禁 |
| **风险提示** | 能识别高危操作并提示用户 |
| **阻断能力** | 能阻止不合规操作（如果配置了） |

### 4.4 记忆能力

| 能力 | 说明 |
|------|------|
| **用户记忆** | 记住用户习惯、偏好、常用工具 |
| **项目记忆** | 记住项目里程碑、重要事件、团队结构 |
| **系统记忆** | 记住系统架构、历史问题、配置偏好 |
| **时间线** | 记住项目生命周期、发布时间线 |
