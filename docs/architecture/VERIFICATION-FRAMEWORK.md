## 一、五 层验证框架

### 信任链问题

```
传统开发: 意图 → 手写代码 → 代码就是意图的忠实表达 (信任链完整)
AI 开发:   意图 → 自然语言 → AI 理解 → AI 生成代码 (3 个环节可能出错)
           ① 说出来的 ≠ 想的 (表达偏差)
           ② AI 理解的 ≠ 说的 (理解偏差)
           ③ AI 写的 ≠ AI 理解的 (实现偏差)
```

### 分层验证体系

```
Layer 5: 业务语义验证     "这个业务流程的结果是对的吗"        (最重要，最难自动化)
       ▲
Layer 4: 场景覆盖验证     "所有业务场景都覆盖到了吗"
       ▲
Layer 3: 行为契约验证     "每个步骤的输入输出符合预期吗"
       ▲
Layer 2: 代码质量验证     "代码本身有没有 bug/安全问题"
       ▲
Layer 1: 可运行性验证     "能不能跑起来/有没有报错"        (最基础，最容易自动化)
```

## 二、各层详解

### Layer 1: 可运行性验证

| 检查项 | 自动化 | 说明 |
|--------|--------|------|
| 编译/构建 | ✓ 自动 | 编译失败立即反馈 |
| 依赖完整性 | ✓ 自动 | 检测不存在的库/API |
| 基础冒烟 | ✓ 自动 | 用默认输入跑一遍 |

### Layer 2: 代码质量验证

| 检查项 | 自动化 | 说明 |
|--------|--------|------|
| 静态分析 | ✓ 自动 | 类型安全、空指针、异常处理 |
| 安全扫描 | ✓ 自动 | 注入攻击、敏感信息、不安全依赖 |
| 代码规范 | ✓ 自动 | 风格一致性、命名、模式匹配 |

### Layer 3: 行为契约验证

**核心概念**：用契约约束 AI 生成的行为

```
契约 = 给定什么输入 → 应该产生什么输出 → 副作用是什么

例: 发送验证码
  输入: 手机号
  前置条件: 手机号格式合法
  输出: 发送成功/失败
  副作用: 验证码写入缓存，有效期 5 分钟
  约束: 同一手机号 60 秒内不可重发
```

**契约来源**：

| 来源 | 说明 |
|------|------|
| 从对话中自动提取 | 用户: "验证码 5 分钟过期" → 契约约束 |
| 从业务规则库继承 | 系统已有"鉴权"规则模板自动应用 |
| 用户主动补充 | 系统提示"涉及资金操作是否需要风控规则?" |

### Layer 4: 场景覆盖验证

**核心问题**：所有业务场景都覆盖到了吗

**三种发现遗漏的方法**：

| 方法 | 说明 |
|------|------|
| 流程图展开 | 每个节点 → 正常路径 + N 种异常路径 |
| 领域知识库 | 内置常见业务领域的风险场景清单 |
| 变异测试 | 主动修改输入/状态来发现弱点 |

### Layer 5: 业务语义验证

| 方法 | 说明 |
|------|------|
| 端到端业务回放 | 用真实/模拟数据跑完整业务故事 |
| 不变量检查 | 定义全局业务约束，任何操作不可违反 |
| AI 交叉审查 | 独立 AI 审查生成 AI 的产出 |

## 四、可视化之间的导航关系

```
宏观 (总览) → 中观 (流程/契约/覆盖) → 微观 (链路) → 底线 (不变量) → 兜底 (审查)

每一层都可以"点击下钻"到更细节的层面
每一层发现问题都可以"一键修复"或"打回迭代"
```

## 五、核心验证机制总结

| 机制 | 本质 |
|------|------|
| 意图澄清 | 将"没想到"的风险从人的盲区转移到系统的检查清单 |
| 行为契约 | 将自然语言的模糊性转化为可机器验证的精确规格 |
| 场景穷举 | 系统化地发现遗漏，而非依赖人的经验 |
| 不变量守护 | 为业务设定"红线"，越线即报警 |
| 交叉审查 | 用另一个视角来发现单一视角的盲点 |

## 六、技术实现对应

| 可视化 | 实现方式 |
|--------|----------|
| 业务流程图 | 从 Intent 对话中自动提取节点和分支，渲染为 DAG 图 |
| 契约验证矩阵 | 契约 × 场景 二维表格，实时更新 |
| 场景覆盖热力图 | 按类别统计覆盖率，风险等级颜色映射 |
| 执行链路追踪 | 类似 OpenTelemetry 的分布式追踪 |
| 不变量守护面板 | 可执行断言，每次操作后自动校验 |
| 验证总览仪表盘 | 聚合所有可视化状态 |
| AI 交叉审查 | 独立 AI 模型审查生成模型产出 |
