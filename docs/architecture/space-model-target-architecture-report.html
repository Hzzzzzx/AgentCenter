Target Premise

目标态不是推翻 MVP，而是把“工作项作业空间”放进多层边界和分层记忆里。

MVP 的内核已经确定：任务列表承载多个工作项，每个工作项拥有独立作业空间。目标架构要做的是把这个内核放进多人协作场景里：不同用户默认隔离，确认为共享的产物和记忆再进入项目共享层。

空间内核 任务列表、工作项、独立作业空间。
默认隔离 用户、工作项、个人记忆先各自独立。
确认共享 产物和记忆确认后进入项目共享层。
治理闭环 确认、复审、淘汰、替代。
02 Target Architecture

目标架构图：同项目多人协作，默认隔离，确认后共享。

目标态和 MVP 的区别不只是多套一层企业边界，而是要回答“不同用户在同一个项目里处理不同工作项时，什么彼此隔离，什么可以共享”。答案是：工作项作业空间和个人记忆默认隔离；被确认的产物、规则和记忆进入项目共享层，再按权限给其他用户复用。

03 Dimensions

目标态不是单一层级，而是四个维度叠加。

如果只画成一棵树，很容易误解。目标架构更像四个维度一起定位一次作业：它属于哪个空间、由谁操作、什么可以共享、能使用哪些记忆。

空间维度

它在哪里

企业、工作空间、项目、任务列表、工作项作业空间，决定作业归属边界。

身份维度

谁在操作

同项目下的不同用户先按自己的可见范围工作，个人上下文默认不暴露给其他人。

共享维度

什么能共享

只有被确认共享的阶段产物、结论、规则和记忆，才进入项目共享层。

记忆维度

能想起什么

当前作业、个人、项目、组织的记忆按边界召回；个人记忆隔离，项目记忆共享。

04 Layered Memory

分层记忆：工作项记忆归工作项空间，不归处理它的个人。

用户 A 在项目里处理工作项 A 时，会产生两类不同记忆：个人偏好、表达习惯、私人上下文归用户 A；需求理解、阶段结论、执行证据、待确认事项归工作项 A 的作业空间。工作项记忆可以被有权限的同工作项协作者复用，但不会自动跨工作项共享；要跨工作项复用，必须确认后进入项目记忆。

当前作业记忆

归属工作项

保存当前工作项的需求、阶段结果、临时判断和待确认事项，可被同工作项协作者和工作流节点复用。

个人记忆

归属用户

保存个人偏好、写作方式、工具习惯和私人上下文，不因为同项目协作而自动暴露。

项目记忆

跨工作项共享

保存项目约定、领域结论、被确认的设计和团队可复用产物，来自工作项记忆的确认共享。

组织记忆

治理后提升

保存跨项目规范、合规要求、通用流程和长期知识资产，需要更严格的复审和淘汰机制。

05 Memory Governance

记忆治理：先留在原空间，确认后再共享，之后持续复审。

分层记忆如果没有治理，会污染上下文。目标态里，工作项记忆和个人记忆都可能产生候选内容，但它们的原始归属不同：工作项记忆先留在工作项作业空间，个人记忆先留在用户个人空间。只有经过确认共享，才进入项目共享层，并且必须能被复审、淘汰或替代。

06 Roadmap

演进路线：先稳空间内核，再加身份和记忆，最后加企业治理。

目标态很完整，但实施仍应该分阶段，避免第一版被复杂组织和记忆系统拖住。

MVP

工作项空间内核

任务列表包含工作项，工作项拥有独立作业空间，第一版可以先采用较轻的共享策略。

V1

用户与默认隔离

加入用户视角、个人记忆隔离、工作项可见范围和当前作业记忆沉淀。

V2

项目共享层

加入确认共享、项目记忆、共享产物复用和跨工作项协作规则。

V3

租户与组织治理

加入组织级记忆、跨团队隔离、审计、复审、淘汰和替代机制。
