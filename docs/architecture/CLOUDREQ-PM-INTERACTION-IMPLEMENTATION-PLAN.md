# CloudReq 项目管理交互边界

> 状态：收敛为薄壳方案。本文只记录主干边界，不规定企业内部接口形态。

## 目标

项目管理页保留面向用户的 CloudReq 语义：

- 自定义项目名称
- CloudReq 项目
- 空间
- 迭代
- 保存选择
- 同步数据

界面展示业务名称，不展示内部 ID。

## 主干边界

AgentCenter 主干只负责：

- 渲染项目管理壳子。
- 保存当前选择。
- 触发同步。
- 将 Provider 返回的标准快照写入通用表。
- 让首页卡片、工作项列表、标题栏和任务编排跟随当前选择刷新。

AgentCenter 主干不规定：

- 企业内部接口路径。
- 企业内部参数名。
- 鉴权、分页、增量同步、权限过滤方式。
- 企业内部错误码和重试策略。

## 企业适配边界

企业内部只需要增量实现 `ProjectDataProvider`，把自己的数据源映射为：

- `ProjectContextDto`
- `ProjectContextOptionsDto`
- `ProjectProviderWorkItemDto`

企业特有字段放入 `extraJson`。确实需要成为主干能力时，再以新增可选字段或新增扩展接口的方式进入主干。

## 低冲突原则

- 前端项目管理页面不因企业内部接口变化而修改。
- 通用同步 Service 不写企业源特例分支。
- 企业内部适配尽量新增文件，不改主干文件。
- 主干文档只描述边界和最小契约，不写企业内部实现步骤。
