AgentCenter Architecture Note

OpenCode Workflow 扩展点地图

这份图解说明 OpenCode 现有的插件、命令、Agent、Tool、Hook、Session 扩展点，以及 Agent Workflow 插件应当接入哪些位置。目标是做外挂式集成：不改 OpenCode 主链路，通过插件和工具把 workflow 能力挂进去。

核心判断 OpenCode 已经有真实子 Agent 机制：task tool 会创建 child session。
最佳接入 Workflow 应做成 OpenCode plugin：注册 commands、tools、agents 和 hooks。
不建议 第一版不要改 LLM loop、message schema、tool registry 源码，也不要先做复杂 UI。

1. OpenCode 原有扩展点总览

OpenCode 的可插入点主要分四层：人类入口层、Agent 上下文层、工具执行层、运行时事件层。 Workflow 插件要尽量接这些现成点，而不是改内部主循环。

2. Workflow 插件准备插入哪些点

插件不是替换 OpenCode，而是在现有扩展点旁边加一个 workflow runtime。命令负责用户入口， plugin tools 负责让 Agent 操作 workflow，hooks 负责状态注入、证据记录和高风险门禁。

建议接入

3. 多子 Agent 的真实运行方式

Workflow 不是让主 Agent 在一段 prompt 里扮演多个角色。OpenCode 侧可以用原生 task 工具创建 child session。每个 Agent 节点都会变成一个真实子会话，带独立上下文、权限和返回结果。

4. Hook 和 Tool 分工

Hook 不做主编排，Tool 不做长期状态源。Workflow core 才是 run 状态源。 Hook 只在关键时刻做拦截、注入、记录和恢复。

责任边界
1

Command

给人入口。比如 /workflow-create 把自然语言流程变成可确认 spec。

2

Plugin Tool

给 Agent 的手。比如 workflow_run、workflow_gate_decide。

3

OpenCode task

真正创建子 Agent。每个 agent node 映射为一个 child session。

4

Hooks

做门禁和记录。工具前拦截，工具后记录 evidence，compaction 时补状态。

5. 影响面矩阵

下面是插件接入后会影响的 OpenCode 点。第一版优先走可逆、外挂式改动，避免进入 OpenCode core。

Impact
OpenCode 点位 Workflow 参与方式 影响 是否改 core
config hook 动态注册 workflow agents、commands、默认配置。 Agent 列表和命令列表多出 workflow 入口。 否

6. 第一版落地顺序

先让 workflow 在 OpenCode 里跑通，再考虑可视化编辑器。第一版重点是自然语言生成 spec、真实子 Agent 执行、状态可恢复。

V1 Plan

用户入口

  1. 安装或启用 agent-workflow OpenCode plugin。
  2. 用户输入 /workflow-create 加自然语言流程描述。
  3. 插件生成 workflow spec，要求用户确认。
  4. 用户输入 /workflow-run 执行一个目标任务。

运行时链路

  1. workflow_run 创建 run 和 checkpoint。
  2. Agent node 调 OpenCode task tool 创建子会话。
  3. Hook 记录工具结果和 evidence。
  4. Gate 判断继续、返工、阻塞或结束。

设计原则：Workflow Core 管状态和秩序，OpenCode task 管子 Agent 执行，Hooks 管拦截和证据。 这样既能做出 Claude Code workflow 类似的效果，又不把 AgentCenter 绑死在 OpenCode 内核里。

Source notes: OpenCode plugin hooks 来自 packages/plugin/src/index.ts； command 加载来自 packages/opencode/src/command/index.ts； 子 Agent 机制来自 packages/opencode/src/tool/task.ts； workflow 目标边界参考 docs/agentcenter/target-capability-architecture.md。
