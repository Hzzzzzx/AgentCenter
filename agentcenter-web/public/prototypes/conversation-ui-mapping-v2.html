进工具折叠块，保留命令、输出、状态、复制。

进 Answer segment，作为用户主要阅读对象。

不展示；只允许公开摘要或调试授权视图。

当前缺口

丢失
file/patch/snapshot/retry/compaction/subtask/agent/todo/session.diff。
混淆
tool 被压成 Skill，metadata、input、attachments 丢失。
噪声
状态事件独立成卡片；应成为 turn 内的 activity rail。

建议的 UI 映射层级

Answer Segment

承载 text/final，可流式更新；只放用户需要读的结论、说明、下一步。

Activity Segment

承载 reasoning_summary、tool lifecycle、status、retry；默认压缩，必要时展开。

Decision Segment

承载 permission/input/confirmation；问题 tab 在标题区，内容区只放问题和选项。

Evidence Segment

承载 file、patch、snapshot、artifact、diff、copy raw；从调试到审查都有证据。
