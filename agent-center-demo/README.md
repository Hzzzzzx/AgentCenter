# AgentCenter Demo

对话驱动的 DevOps 智能中枢演示。

## 当前首页

当前首页是白色网页端工作台，采用左中右三栏布局，并让中心对话工作台成为主视觉。

> **接手提示**：这里写的是 2026-04-27 刚修改完成并推送到 `main` 的新版首页。其他 agent 如果找不到页面，请优先打开 `client/index.html`，并对照 `../docs/prototype/homepage.html` 与 `tests/homepage-workbench.spec.js`。

| 区域 | 当前内容 |
|------|----------|
| 顶部导航条 | 支持展开/收起，包含全局搜索、通知、设置和用户角色 |
| 顶部指标面板 | 默认收起，展开后显示今日部署、构建成功率、活跃告警、Sprint 进度 |
| 左侧栏 | 默认展开会话列表；会话列表、角色切换、工作台、工具链、智能体状态均可折叠 |
| 中心区域 | 对话工作台，支持输入指令和示例场景动态对话 |
| 右侧栏 | 上下文详情、主动预警、智能体协作三类视图 |
| 底部历史面板 | 默认收起，展开后显示最近活动、全流程历史和执行记录 |

## 重置演示状态

重置会将故事状态、对话历史、角色重置为初始状态。

**HTTP 方式：**
```bash
curl -X POST http://localhost:4000/reset
```

**Socket.io 方式：**
```javascript
socket.emit('reset');
socket.on('demo_reset', (data) => console.log(data));
// → { status: 'ok', activeRole: 'management' }
```

## 演示脚本（Presenter Script）

### 故障处置故事（面向管理层 + 工程团队）

各位好，今天我们演示一个真实的故障场景。下午两点，系统突然收到大量告警，提示身份认证服务出现异常。我们的 AI 助手立即捕获这一情况，在气泡墙上呈现关键指标变化——服务响应时间飙升、错误率突破阈值。管理层通过仪表盘看到的是全局影响范围，而值班工程师可以直接看到具体是哪个微服务出现问题。AI 自动分析了关联日志，定位到是最近一次部署导致的配置变更。在多方确认后，工程师通过对话界面一键发起回滚，整个故障恢复时间从传统的 45 分钟缩短到 8 分钟。最后系统自动生成了故障报告，管理层可以在仪表盘上看到本次事件的完整时间线和资源消耗。

### 发布流程故事（面向管理层）

现在我们演示一个完整的发布流程。首先产品经理在系统中提交了一个新功能的上线申请，系统自动通知到研发负责人进行技术评审。研发团队完成代码审查后，AI 自动触发了构建流水线，我们可以在气泡墙上实时看到构建进度。测试团队通过对话查询构建结果，确认无误后直接用自然语言发起部署审批。管理层在仪表盘上可以看到所有待审批的发布任务，只需要一句话就能完成批量确认。最终部署平台自动将服务发布到预生产环境，整个流程从申请到上线全程可追溯，管理层无需逐个电话确认进展。

### 需求流程故事（面向工程团队）

这个演示展示一个端到端的需求交付流程。产品经理在需求管理平台创建了新功能的需求文档，AI 自动解析了需求内容并生成了技术实现建议。研发负责人通过对话界面确认了实现方案，系统自动拆解出了具体的开发任务并分配给相应成员。开发过程中，AI 持续追踪代码评审进度和构建状态，发现阻塞时自动提醒相关人员。测试团队通过对话直接发起回归测试，测试结果自动关联到原始需求。最后质量团队通过仪表盘查看了需求的完整测试覆盖率，AI 自动生成了上线前的质量评估报告。

## MVP 局限性

本 Demo 为演示版本，存在以下限制：

| 限制 | 说明 |
|------|------|
| **Mock 模式** | 所有外部调用均为模拟，无真实工具链集成 |
| **仅桌面端** | 演示界面针对桌面浏览器优化，移动端体验受限 |
| **无认证** | 无用户认证、权限管理机制 |
| **无持久化** | 所有数据存储在内存，服务器重启后丢失 |
| **无真实部署** | 部署命令仅模拟执行，不触发真实环境变更 |
| **仅供演示** | 不适合作为生产环境参考架构 |

**当前为 Mock 演示模式，所有外部调用均为模拟。**
